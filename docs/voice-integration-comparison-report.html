The Full Voice I/O Loop

Speak → Handy transcribes → agent-meow receives → Hermes executes → Voicebox speaks the response. All local, all offline.

sequenceDiagram
    actor User
    participant Handy as Handy STT
    participant Meow as agent-meow
    participant Hermes as Hermes Agent
    participant Voicebox as Voicebox TTS

    User->>Handy: Press hotkey, speak
    Handy->>Handy: VAD + Whisper/Parakeet
    Handy->>Meow: Paste transcript to composer
    Meow->>Meow: POST /v1/sessions/<id>/events
    Meow->>Hermes: HermesExecutor.run_turn()
    Hermes->>Hermes: computer_use / shell / skills
    Hermes->>Meow: TextChunk + TurnComplete
    Meow->>Meow: Session stream → SSE → Web UI
    Meow->>Voicebox: speak tool → POST /speak
    Voicebox->>Voicebox: TTS engine + voice profile
    Voicebox->>Meow: audio_url + generation_id
    Meow->>User: AudioBlock inline player

Implementation Status

What's code-ready vs what needs wiring. Most of the stack was already built — we only added the Voicebox TTS path.

Handy --transcribe-file — fully implemented in Handy main. Outputs JSON with text, model, rtf. Headless mode bypasses GUI.
_execute_voice_tool — fully implemented in tool_dispatch.py. Handles Handy STT + Voicebox/VibeVoice TTS. 11 tests passing.
_VOICE_TOOLS frozenset — all 4 tool names in _ALL_LOCAL_TOOLS + _NATIVE_RELAY_BUILTIN_TOOLS. Dispatch routing at line ~5901.
HermesExecutor — agent-meow drives Hermes as a harness with session resume (--resume session_id) + policy bridge (pre_tool_call hook).
_EXTERNAL_ASSISTANT_MESSAGE_TYPE — sessions.py has built-in mechanism for landing external agent results into the workspace transcript.
AudioBlock.tsx + parseAudioFromToolOutput() — component exists, parses audio_url from JSON tool output. NOT yet wired into ToolCard.
AudioBlock → ToolCard wiring — Task 4: ~10 lines in ToolCard.tsx to call parseAudioFromToolOutput before rendering raw text.
Voicebox MCP server config — Task 1: ~10 lines YAML for stdio transport MCP server registration.
voicebox-agent example spec — Task 2: ~30 lines YAML example agent with Voicebox MCP + Handy STT tools.

Remaining Gaps vs Cherry

Explicitly deprioritized per design decisions. Listed here for completeness.

W

WeChat integration

Hermes has 5 messaging platforms (Telegram, Discord, Slack, WhatsApp, Signal). WeChat would require a new gateway adapter — China-market specific.

Not built
A

App store surface

Cherry has a dedicated app store UI. Our stack uses shell/computer_use (winget/choco) — functional but less polished as a consumer surface.

Not built
D

PC diagnostics

Cherry's 电脑医生 monitors system errors, network, drivers, device health. We deprioritized this — Hermes could add it as a skill (cron + WMI/EventLog).

Deprioritized
